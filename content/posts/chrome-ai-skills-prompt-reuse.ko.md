## 프롬프트를 다시 붙여넣어야 하는 일에서 벗어나는 기능

이런 경험이 있으신가요? 코드 리뷰를 위해 AI 도구를 사용할 때마다 "전통 중국어를 사용하고, CRITICAL/HOLD/MEDIUM 문제를 나열하세요..."라는 지침을 다시 붙여넣어야 한다면?

이것이 바로 **Chrome AI 스킬** — Google's 새 기능 — 이 해결하고자 하는 문제입니다.

간단합니다: 자주 사용하는 프롬프트를 '스킬)로 저장하고 이름을 붙인 다음, AI 입력 필드가 있는 모든 웹페이지에서 해당 스킬을 직접 소환할 수 있습니다 — 다시 입력하거나 복사/붙여넣기할 필요가 없습니다.

작은 기능처럼 보이지만, 그 이후에 있는 추세는 심각하게 받아들여야 할 가치가 있습니다.

---

## Chrome AI 스킬은 어떻게 작동하나요?

Google의 문서에 따르면, Chrome AI 스킬의 핵심 로직은 매우 간단합니다:

1. **스킬 만들기**: Chrome 설정에서 프롬프트를 추가하고 이름을 붙입니다 (예: "코드 리뷰", "영어로 번역")
2. **스킬 호출하기**: 모든 AI 채팅 인터페이스(Gemini, ChatGPT, Claude 웹 버전 등)에서 단축키를 사용하여 저장된 스킬 목록을 불러옵니다
3. **한 번의 클릭으로 삽입**: 하나를 선택하면 프롬프트가 입력 필드에 자동으로 채워집니다 — 보내기만 하면 됩니다

여기서 핵심은 "**사이트 간 범용**"입니다 — 어느 AI 플랫폼을 사용하든 Chrome을 사용하는 한, 스킬 라이브러리가 따라옵니다.

---

## 다른 플랫폼의 스킬 기능과 비교

"스킬"은 AI 업계에서 새로운 용어가 아니지만, 각 플랫폼은 이를 매우 다르게 정의합니다.

### Claude Code 스킬: 개발자의 워크플로우 스크립트

Claude Code(Anthropic의 CLI 도구)에도 스킬 메커니즘이 있지만, 성격은 완전히 다릅니다.

Claude Code 스킬은 본질적으로 **실행 가능한 워크플로우 정의**입니다:

```markdown
# /commit skill

Run git commit 프로세스:
1. staged files 확인
2. diff 기반으로 commit message 자동 생성
3. 확인 후 commit 실행
```

`/commit`, `/review-pr`, `/deploy`와 같은 명령어로 스킬을 호출할 수 있으며, 텍스트 삽입만 아니라 전체 작업 세트를 실행합니다. 차이점: **Chrome AI 스킬은 프롬프트 템플릿이고, Claude Code 스킬은 실행 가능한 프로그램입니다**.

### OKX 에이전트 스킬: 트레이딩 AI의 기능 단위

OKX의 에이전트 스킬은 또 다른 차원입니다. 암호화폐 트레이딩 에이전트 프레임워크에서 "스킬"은 AI 에이전트가 호출할 수 있는 **기능 모듈**을 나타냅니다:

- `market_analysis_skill`: 현재 시장 조건 분석
- `risk_assessment_skill`: 포지션 리스크 평가
- `execute_trade_skill`: 트레이딩 주문 실행

여기서 "스킬"은 소프트웨어 엔지니어링 모듈 개념에 더 가깝습니다 — 에이전트가 요구에 따라 결합하고 호출할 수 있는 기능 단위입니다.

### 세 스킬의 포지션 비교

| 기능 | Chrome AI 스킬 | Claude Code 스킬 | OKX 에이전트 스킬 |
|---|---|---|---|
| 대상 사용자 | 일반 사용자 | 개발자 | 트레이딩 AI 개발자 |
| 실행 수준 | 프롬프트 텍스트 | 워크플로우 | AI 에이전트 기능 모듈 |
| 크로스 플랫폼 | ✅ 사이트 간 | ❌ CLI 전용 | ❌ OKX 생태계만 |
| 자동화 수준 | 낮음(수동 트리거) | 중간(반자동 워크플로우) | 높음(에이전트 자율 호출) |

## 이 추세가 주목할 가치가 있는 이유

세 개의 다른 플랫폼, 세 개의 다른 스킬 구현이지만, 그 이후에는 모두 같은 문제가 있습니다: **AI를 사용하는 것이 너무 비용이 듭니다**.

금전적 비용이 아닙니다 — **인지 비용**.

AI를 사용할 때마다: 최적 프롬프트 형식을 떠올리고 → 다시 입력 → 조정 → 다시 입력해야 합니다. 이 마찰 때문에 많은 사람이 "알지만 매번 신중하게 프롬프트를 작성하는 것이 귀찮다"고 합니다.

스킬 시리즈의 본질은 이 마찰을 줄이는 것입니다:

- **Chrome AI 스킬**: 일반 사용자의 재사용 장벽을 낮춤
- **Claude Code 스킬**: 개발자가 모범 사례를 반복 가능한 실행으로 코딩할 수 있게 함
- **에이전트 스킬**: AI 시스템이 기능 모듈을 모듈화하여 결합할 수 있게 함

개인 도구에서 에이전트 아키텍처까지, 모든 사람이 같은 방향으로 나아가고 있습니다: **'AI를 사용하는 스킬 자체'를 저장 가능하고 공유 가능하며 재사용 가능한 자산으로 전환**.

## 개발자들은 지금 무엇을 할 수 있나요?

Chrome AI 스킬이 출시되기를 기다릴 필요 없습니다 — 유사한 워크플로우 최적화는 지금 바로 할 수 있습니다:

### 1. 프롬프트 라이브러리 구축

모든 메모 도구(Notion, Obsidian, 일반 텍스트라도)를 사용하여 자주 사용하는 프롬프트를 분류하고 저장하세요. Chrome AI 스킬이 출시되면 매끄럽게 마이그레이션할 수 있습니다.

저장할 만한 일반적인 유형:
- 코드 리뷰 스타일 지침(언어/형식/출력 구조)
- 문서 번역 템플릿(전통 중국어/간체/영어 어조 요구사항)
- 디버깅 문제 설명 프레임워크(오류 메시지 + 예상 동작 + 시도한 솔루션)
- API 문서 생성 형식

### 2. Claude Code 스킬로 워크플로우 자동화

이미 Claude Code를 사용하고 있는 경우, `.claude/` 디렉토리의 스킬 정의는 프롬프트 템플릿보다 더 강력합니다. 반복적인 워크플로우 — 커밋, PR 리뷰, 배포 확인 — 를 스킬로 작성하고, `/command`로 직접 호출하세요.

### 3. AI 스킬 표준화 추세 주시

Chrome의 움직이는 "프롬프트 관리"가 개인 습관에서 플랫폼 기능으로 진화하고 있습니다. 다음 단계는 크로스 디바이스 동기화, 스킬 공유 마켓플레이스, API 액세스가 될 수 있습니다.

개발자들에게는 지금 개인 스킬 자산을 구축하기에 좋은 시기입니다 — 어떤 플랫폼을 사용하든 상관없이.

## 마무리

Chrome AI 스킬 자체는 "매번 프롬프트를 다시 입력해야 하는"라는 실제 pain point를 해결하는 실용적인 작은 기능입니다. 하지만 더 넓은 그림에서, 이것은 AI 도구 표준화의 신호입니다: **AI의 인터페이스 레이어가 성숙해지고 있습니다**.

프롬프트 템플릿(Chrome) → 워크플로우(Claude Code) → 에이전트 기능 모듈(OKX)에 이르기까지, 스킬이라는 개념이 추상 사다리를 오르고 있습니다.

개발자들이 지금 해야 할 일은: 이 추세가 고정되기 전에, 가장 가치 있는 AI 사용 패턴을 축적하는 것입니다 — 그것이 스킬, 템플릿, 또는 다른 어떤 이름이든간에요.
