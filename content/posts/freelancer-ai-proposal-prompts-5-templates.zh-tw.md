## 73%的自由工作者都在用AI，但時薪沒漲

Useme在2026發佈的自由工作者報告指出，超過七成的接案者已經把AI納入日常工作流程[UNVERIFIED]。但同一份報告提到一個尷尬的事實——用AI的人，時薪平均並沒有比不用的人高。只有一小群被歸類成「AI賦能者」的接案者拿到接近四成的時薪溢價[UNVERIFIED]。

差別在哪？不是工具，是prompt。

業界普遍觀察到一件事，大家都打開ChatGPT、Claude，但九成的人只丟「幫我寫一封提案信」這種句子進去，拿到的就是長得一模一樣、客戶看一眼就知道是AI寫的、罐頭味十足的版本。剩下那一成，把prompt當作系統工程在設計，輸出就是另一個世界。

接案這件事，從提案、報價、催款、開發新客戶到結案，每一個環節都可以被prompt模板化。下面5個是Judy AI Lab觀察到接案社群最常被反覆改良、最有實戰價值的模板。直接複製可用。

## 模板1：客製化提案信

罐頭提案信跟客製化提案信，回覆率差很多。關鍵在於把對方產業、痛點、報價結構當變數塞進去。

```
你是擁有10年經驗的[自由工作者領域]接案專家。
請幫我寫一封繁體中文提案信給以下客戶：

- 客戶產業：[填產業]
- 客戶痛點：[填具體痛點，例：流量停滯、轉換率低]
- 我的核心優勢：[填2-3個具體成果，含數字]
- 預算範圍：[填區間]
- 交付時程：[填週數]

語氣要：專業但有溫度，不卑不亢，前3句必須講中對方痛點。
結尾留一個低門檻的下一步（不是逼簽約）。
```

關鍵在最後兩行的語氣指令。沒寫這兩行，AI預設會用很「業務」的口吻，客戶秒刪。

## 模板2：一次生3個報價方案

報價單只給一個數字最容易被殺價。給3個方案，客戶心理會自動跑「比較」而不是「砍價」。

```
請根據以下案件需求，生成3個報價方案：

- 案件類型：[填]
- 核心交付物：[填]
- 我的成本底線：[填]

請輸出：
1. 精簡版（核心需求，最低價）
2. 標準版（含1次修改+基本售後）
3. 旗艦版（含3次修改+30天保固+數據追蹤）

每個方案請列：
- 交付項目（具體可驗證）
- 時程
- 價格
- 適合誰選

結尾加一句話引導客戶往「標準版」靠。
```

這個輸出的關鍵不是數字漂亮，是「適合誰選」那一句。它把選擇焦點從價格轉移到「我是哪種客戶」。

## 模板3：催款信（3種語氣一次生）

催款是接案最尷尬的環節。一次跟AI要3個語氣版本，當下用哪個自己挑。

```
請幫我寫3個版本的催款訊息給以下客戶：

- 案件名稱：[填]
- 應付金額：[填]
- 逾期天數：[填]
- 過往合作關係：[填，例：合作過2次/初次合作]

3個版本分別是：
A. 溫和提醒版（彷彿幫對方想起）
B. 正式商務版（含付款截止日）
C. 最後通牒版（提及後續法律或終止合作）

每個版本300字以內，繁體中文，台灣商務口吻。
```

催款不是寫不出來，是當下心情亂寫會把關係搞砸。預先生成、平常存著，需要時直接挑。

## 模板4：cold email開發信

陌生開發信打開率業界普遍偏低。要拉到雙位數，前12個字，也就是信件主旨加第一句，就要打中。

```
請幫我寫一封cold email給[目標公司]：

- 目標公司產業：[填]
- 我觀察到他們的問題：[填，必須是公開可驗證的，例：官網載入慢、社群停更3個月]
- 我能提供的解法：[填，一句話]
- 信件主旨：限15字內，要有好奇心缺口

信件結構：
- 第1句：點出我觀察到的具體問題（不能說「我看到貴公司網站」這種廢話）
- 第2-3句：用一個類似客戶的成果背書
- 第4句：低門檻邀請（不是「方便聊聊嗎」，是「我做了一個10秒的audit影片可以寄你」）

整封信不超過150字。
```

關鍵是把「方便聊聊嗎」換成具體可驗證的下一步。前者的回覆率在業界一直被詬病偏低。

## 模板5：結案信順手要testimonial

結案那一刻是要推薦信、要回購、要轉介紹的黃金時間。但大部分人結案信只寫「謝謝合作」就結束。

```
請幫我寫一封結案信，包含3個目的：

- 客戶名稱：[填]
- 案件成果（含數字）：[填]
- 合作期間：[填]

信件要做到：
1. 開頭：感謝+具體成果回顧（不是空泛感謝）
2. 中段：自然邀請對方寫一段50-80字的testimonial，給3個提示問題讓他填空就好
3. 結尾：留一個「如果有朋友也需要...」的轉介紹鉤子，但不要太業務

繁體中文，溫度要夠但不油膩。
```

中段的「填空式testimonial」是關鍵。直接問「可以寫一段推薦嗎」十有八九拿不到，給3個問題讓對方挑著答，回收率明顯不一樣。

## Claude跟ChatGPT在這5個模板上的差異

實務上把同一組prompt丟給Claude Sonnet 4.6跟ChatGPT GPT-5.2，輸出風格有明顯差別。Claude在「語氣拿捏」和「不卑不亢的商務溫度」上表現相對細膩；ChatGPT在「結構工整」跟「條列清楚」上比較順手[UNVERIFIED]。

業界普遍觀察到的搭配方式是——提案信、催款信、結案信這類「人味重」的場景用Claude；報價單、cold email這類「結構重」的場景用ChatGPT。當然這只是一個粗略分類，每個人案件型態不同，最後還是要自己試一輪才知道。

## 為什麼這5個模板存在的意義不是「省時間」

省時間只是表面。真正的差別是——你接案的每一個觸點，都有一個事先想清楚的版本在那邊等你。

不會發生「客戶突然來信、我手忙腳亂亂回」這種事。也不會發生「催款催不下去最後乾脆放掉」這種事。

工具是公平的，prompt是不公平的。

七成的人用同一個工具，一成的人賺到溢價。差的就是有沒有把這些重複場景當作系統去設計。

以上是一個觀察。
