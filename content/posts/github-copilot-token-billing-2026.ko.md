## 6월부터 GitHub Copilot 청구 방식이 달라집니다

GitHub Copilot을 사용하고 계시다면, 5월 말까지 확인해야 할 사항이 있습니다.

GitHub은 **2026년 6월 1일부터** 모든 Copilot 플랜이 고정 구독에서 "AI 크레딧 사용량 기반 과금"으로 전환된다고 밝혔습니다 — 즉, 사용한 만큼, 토큰당 지불합니다.

구독료 자체는 오르지 않지만, 전체 청구 "로직"이 변경되었습니다. 개발자들은 말합니다: "같은 돈을 내지만, 더 적게 받을 수 있다" —이는 과장이 아닙니다 —실제 사용 방식에 따라 다릅니다.

이 글에서는 가장 중요한 사항들을 정리해 드립니다.

---

## 왜 변경하나요? AI 크레딧이란?

이전에는 Copilot의 유료 기능(채팅, 코드 리뷰 등)은 "프리미엄 요청 단위(PRU)"로 계산되었습니다 —대화의 길이나 모델의 무거운 정도에 관계없이 각 작업은 고정된 수의 PRU를 소모했습니다.

새로운 시스템이 그것을 바꿉니다.

**AI 크레딧은 직접 토큰 사용량에 해당합니다:**
- 입력 토큰 + 출력 토큰 + 캐시된 토큰이 모두 청구에 포함됩니다
- 모델마다 요율이 다릅니다(더 강력한 모델일수록 더 많이 소모합니다)
- **1 AI 크레딧 = $0.01 USD**

GitHub의 CPO 마리오 로드리게스가 이러한 변경의 이유를 설명했습니다: "짧은 채팅 질문이 여러 시간에 걸친 자동 코딩 세션과 동일한 리소스를 소모할 수 있다 —이는 모두에게 공정하지 않습니다."

새로운 시스템은 청구를 실제 컴퓨팅 비용에 가깝게 만들지만, 사용자에게는 이전의 "고정 비용" 예측 가능성이 사라졌습니다.

---

## 각 플랜에는 얼마의 AI 크레딧이 포함되나요?

**좋은 소식: 월 이용료는 오르지 않았습니다.**

| 플랜 | 월 이용료 | 포함된 AI 크레딧 |
|------|------------|---------------------|
| Copilot Pro | $10/월 | $10 크레딧 |
| Copilot Pro+ | $39/월 | $39 크레딧 |
| Copilot Business | $19/사용자/월 | $19 크레딧 |
| Copilot Enterprise | $39/사용자/월 | $39 크레딧 |

각 플랜에는 월 이용료와 동일한 크레딧이 포함됩니다. 다 쓰면, 유료 플랜은 추가 크레딧을 구매할 수 있습니다. 관리자가 사용량 한도를 설정하면, 크레딧이 0이 되면 더 이상 고급 기능을 사용할 수 없습니다.

> **중요:** GitHub은 명시적으로 "대체 기능은 더 이상 제공되지 않습니다"라고 밝았습니다. 이전에는 크레딧이 부족할 때 Copilot이 자동으로 더 저렴한 모델로 전환하여 서비스를 계속했을 수 있습니다. 6월부터는 크레딧이 다 쓰이면 그것으로 끝입니다 —자동다운그레이드 없음.

---

## 무엇이 무료인가요? 무엇이 크레딧을 소모하나요?

가장 중요한 목록입니다.

### ✅ AI 크레딧을 소모하지 않음 (무료)
- **코드 자동완성** —타입하는 실시간 제안, 완전히 무료
- **다음 편집 제안** —다음 편집 위치를 예측, 무료

### 🔴 AI 크레딧을 소모함 (토큰별 청구)
- **Copilot 채팅** —모든 대화마다 청구
- **다단계 코딩 세션** (에이전트 모드) —토큰 사용량이 매우 높을 수 있음
- **코드 리뷰** —6월부터 GitHub Actions 시간도 소모
- **고급 모델을 사용하는 모든 기능** (GPT-4o, Claude 3.7 Sonnet 등)

코드 자동완성 primarily을 사용하는 경우, 영향은 상대적으로 작습니다. 채팅을 많이 사용하거나 Copilot에 장시간 워크플로를 실행시키는 경우, 심각하게 평가해야 합니다.

## 최대한 절약하는 방법: 6월 전에 반드시 해야 할 2가지 단계

### 1. 5월 초에 "청구서 미리 보기" 확인

GitHub은 **2026년 5월 초에** 청구 개요 페이지([github.com Billing Overview](https://github.com))에 "청구서 미리 보기" 기능을 출시하여,정식 시행 전에 새 과금 모델에서의 예상 비용을 볼 수 있게 한다고 밝�습니다.

비용을 들이지 않고 현재 사용 습관이 새 시스템에서 어떻게 비용으로 전환되는지 볼 수 있는 유일한 기회입니다. **5월 말 전에 반드시 확인하세요.**

### 2. AI 학습 데이터 제외 여부 확인

GitHub은 **2026년 4월 24일**에 정책을 조용히 업데이트했습니다: Free, Pro, Pro+ 플랜의 대화 데이터는 기본적으로 AI 모델 학습에 사용되며,actively 제외하지 않으면 기본적으로 활용됩니다.

코드 프라이버시가 중요하다면, 지금 GitHub 설정으로 이동하여 확인하세요. 엔터프라이즈 플랜 사용자는 영향받지 않습니다 —기본값은 학습에 대화 데이터를 사용하지 않는 것입니다.

## 엔터프라이즈 사용자는 완충 장치를 받고, 개인 개발자는 스스로 주시해야 합니다

비즈니스 및 엔터프라이즈 고객에게, GitHub은 3개월간 "전환 지원금"을 제공합니다:

- **비즈니스** 플랜: 6월 1일부터 9월 1일까지 월 1인당 추가 $30 크레딧
- **엔터프라이즈** 플랜: 6월 1일부터 9월 1일까지 월 1인당 추가 $70 크레딧

9월 1일부터는 정상 플랜 할당량으로 복귀합니다.

추가로, 엔터프라이즈 관리자는 더 세분화된 사용량control 도구를 받게 되어, 다음을 설정할 수 있습니다:
- 조직 전체의 크레딧 한도
- 각 비용센터의 한도
- 사용자별 한도

이는 대기업에서 무제한 크레딧 사용으로 회사 청구서가 폭증하는 것을 방지하는 데 매우 유용합니다.

## 이 변경이 당신에게 좋은가요, 나쁜가요?

코드 자동완성을 primarily 의존한다면, 이 변경은 거의 영향이 없습니다 — 자동완성은 무료로 유지됩니다.

채팅, 에이전트 모드를 많이 사용하거나 Copilot에 복잡한 다단계 작업을 실행시킨다면, 주시해야 합니다. $10/월 크레딧은 생각보다 빠르게 소진될 수 있습니다, 특히 더 강력한 모델을 사용할 때.

개발자 커뮤니티의 일반적인 시각은: "같은 돈을 내지만, 더 적게 받을 수 있다"입니다. 그러나 GitHub의 입장은 이것이 청구를 더 공정하게 만든다는 것입니다 —가벼운 사용자가 무거운 사용자의 컴퓨팅 리소스를 보조하지 않게 됩니다.

어찌되었든, **5월 초에 청구서 미리 보기를 확인하는 것이 지금 가장 중요한 일입니다**. 숫자가 나타나기 전에, 모든 우려나 안도 mere speculation일 뿐입니다.

## 자주 묻는 질문 (FAQ)

**Q1: 코드 자동완성이 정말 완전히 무료인가요?**
네. GitHub은 명시적으로 코드 자동완성과 다음 편집 제안이 AI 크레딧을 소모하지 않으며 6월 이후에도 무료로 유지된다고 밝혔습니다.

**Q2: 크레딧이 다 쓰이면 어떻게 되나요?**
고급 기능(채팅, 에이전트 모드, 코드 리뷰)이 작동을 멈춥니다. 6월부터 더 저렴한 모델로 자동 대체하는 기능이 없습니다. 추가 크레딧을 구매하거나 다음 청구 주기 초기화까지 기다릴 수 있습니다.

**Q3: 다른 모델도 동일한 양의 크레딧을 소모하나요?**
아닙니다. 모델이 더 강력할수록(예: GPT-4o, Claude 3.7 Sonnet) 요율이 더 높습니다 —동일한 대화 길이도 더 많은 크레딧을 소모합니다. 모델 선택 시 성능과 비용 사이의 트레이드오프를 고려해야 합니다.

**Q4: 엔터프라이즈 사용자인데 전환 지원금은 얼마나 오래 받나요?**
비즈니스 플랜은 월 1인당 추가 $30 크레딧, 엔터프라이즈는 월 1인당 $70 크레딧을 받습니다. 지원금 기간은 6월 1일부터 9월 1일까지이며, 이후에는 정상 플랜 할당량으로 복귀합니다.

**Q5: AI 학습 데이터 수집에서 제외하려면 어떻게 하나요?**
GitHub 설정 → Copilot → "Allow GitHub to use my interactions" 옵션을 찾아 끕니다. 엔터프라이즈 플랜은 기본적으로 학습에 참여하지 않으므로 추가 조치가 필요하지 않습니다.

**Q6: PRU(프리미엄 요청 단위)는 아직 사용되나요?**
아닙니다. 2026년 6월 1일부터 PRU는 완전히 AI 크레딧으로 대체됩니다. PRU는 더 이상 어떤 청구 계산에도 사용되지 않습니다.
