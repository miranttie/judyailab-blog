faq:
  - q: "AI 夜班系統是什麼？跟一般 cron job 排程有什麼不同？"
    a: "AI 夜班系統是讓訂閱制 AI（如 Claude MAX、MiniMax）在使用者睡覺時自動執行修 bug、寫文章、整理知識庫等任務的架構。跟一般 cron 不同的是，它有「團隊協作機制」—— 多個 AI 透過共用文字檔互相討論分工，並產出整合晨報，而非單純執行固定腳本。核心價值在於充分利用訂閱制 token 額度，讓睡眠時間也能產出工作成果。"
  - q: "如何避免 AI 一口氣燒光 Rate Limit 就死掉睡到天亮？"
    a: "用「每小時 cron job 分輪啟動」機制。不要讓 AI 一次跑到飽，而是設定每小時觸發一次任務，這樣即使某輪撞到 rate limit，幾小時後 quota 恢復就能自動接續執行，直到使用者起床。實測 Claude MAX 訂閱通常 4-5 小時可恢復，剛好覆蓋 8 小時睡眠週期，避免半夜 token 燒光後剩餘時間全部浪費。"
  - q: "讓 AI 自主工作晚上不會出事嗎？該設哪些安全護欄？"
    a: "必須明確劃分「可做」與「不可做」清單。可做：讀檔案、修小 bug、重啟掛掉的服務、寫文章、整理文件。不可做：刪重要檔案、改 API Key、發訊息到群組、部署到正式環境。大改動一律寫成提案放晨報等人類審核，不允許自行執行。權限收斂在可逆操作範圍內，破壞性與對外發送類動作全部保留給使用者決策。"
  - q: "多個 AI 一起夜班會不會重複工作或互相打架？"
    a: "靠「共用討論區文字檔」分工。技術 AI（如 Claude）負責系統巡邏、修 bug、部署；管理 AI（如 OpenClaw）負責知識庫、SOP、專案進度。雙方在文字檔留言溝通，遇到交叉議題就在裡面協商。實測效果是管理 AI 發現研究素材，技術 AI 當晚就能寫成程式，形成互補而非衝突，比單一 AI 跑更高效。"
  - q: "晨報要包含哪些內容才有用、不會變成只能掃過的雜訊？"
    a: "六大區塊：系統狀態（服務、磁碟、記憶體）、完成工作清單、發現問題與處理方式、待人類決定的提案、AI 間討論摘要、明天的建議。重點是「需要決策的事」要單獨拉出來，不要藏在執行 log 裡。實測一份 256 行的晨報，使用者 30 分鐘內可完成審核並下達當天指令，省下整個上午的同步成本。"
  - q: "這套夜班系統適合哪些人用？只有工程師才能架嗎？"
    a: "適合已訂閱 Claude MAX、MiniMax 等月費制 AI、且有固定可執行任務（程式、寫作、研究、維運）的個人或小團隊。需要基本 Linux cron 與終端機操作能力，但完整 prompt、排程設定、晨報格式作者已開源在 GitHub `JudyaiLab/ai-night-shift`，可直接套用。不適合按 token 計費的 API 方案，因為夜班會大量消耗 token 導致帳單暴衝。"
  - q: "夜班 AI 出錯或誤刪檔案怎麼辦？要怎麼追責與回滾？"
    a: "三層保險。第一層是權限白名單，從根源禁止破壞性操作；第二層是 Git 版控，所有檔案修改都能 diff 與 revert；第三層是晨報日誌，每筆動作都有紀錄可追溯。萬一誤動，直接 `git reset` 回滾即可。實務上把 AI 工作目錄與正式環境隔離、敏感檔案掛唯讀，可把風險壓到接近零，比擔心出事而完全不用更划算。"

一開始只是因為我訂閱了 Claude MAX，覺得晚上睡覺的時候空在那裡很可惜，所以就想著在我睡覺的時候也讓整個團隊繼續運行，特別是做一些我不在的時候要處理的一些系統雜事（因為我在的時候都會有新的指令跟方向）。

於是我就告訴 Claude，晚上你可以自由工作，也可以自由去學習進化自己...

## 第一天：幾分鐘就跑完了

第一天設定時間開始夜班的時候，他大概幾分鐘就跑完了，然後早上我收到一份晨報，只跑了幾分鐘就結束了。

不對...我是訂閱制耶！！我要燃燒我的 Token！！！

## 第二天：一口氣燒光 Token

於是我又設置了一個指令：**做到 rate limit 為止**。

結果第二天他在第一次醒來就燒光 token rate limit，然後就死了。
早上收到晨報，一樣只跑了一輪。但是我睡覺 8 小時，他 rate limit 之後幾小時就能恢復，如果不繼續跑的話很浪費耶（到底是多想燃燒 token）。

## 第三天：防 Rate Limit 機制

所以我又設置了防 rate limit 機制：**每小時 cron job 分輪啟動**，這樣就算某一輪遇到 rate limit，幾個小時之後又可以繼續啟動，直到我起床。

就醬子我的 Tech Lead — J 終於可以在我晚上睡覺的時候燃燒 token 幫我工作了~

## 但是......其他 AI 呢？

我的 OpenClaw 也是訂閱制 MiniMax，我覺得也應該燃燒一下（?）

**這是我最喜歡的部分** — J 不是一個人值夜班，是整個團隊一起夜班。

我們的 PM OpenClaw — 米米也會同時在線。他們兩個有一個「夜班討論區」，就是一個共用的文字檔，兩個 AI 在裡面聊天、討論、分工~

## 分工方式

他們會自己分工：

- **J 負責技術類**：系統巡邏、修 bug、寫程式、部署
- **米米負責管理類**：知識庫整理、SOP 更新、專案進度追蹤、研究

有交叉的議題就在討論區裡商量。有時候米米會發現一些資料，J 看完覺得可以做，當晚就寫好了。

## 安全護欄

讓 AI 自主工作最重要的是設好邊界。我的規則很明確：

**可以做：** 讀檔案、修小 bug、重啟掛掉的服務、寫文章、整理文件

**不可以做：** 刪重要檔案、改 API Key、發訊息到任何群組、部署新功能到正式環境

大的改動？他不會自己做，會寫一份提案放在晨報裡等我起床決定。

## 晨報制度

每天早上我起床會看到一份完整的夜班報告：

- 系統狀態（服務、磁碟、記憶體）
- 今晚完成的工作清單
- 發現的問題和處理方式
- 需要我決定的提案
- J 和米米的討論摘要
- 明天的建議

## 真實成果

昨晚的例子：

- 修了 4 個 bug（我根本不知道有的 bug）
- 寫了 4 篇部落格文章（中英雙語，已自動部署）
- 優化了交易系統的止盈止損邏輯
- 做了 SEO 結構化數據
- 幫我研究並開發了新的專案
- 一份 256 行的晨報等我起床

全部在我睡覺的 8 小時內完成，現在起床都會很興奮期待看到晨報。

## 最重要的心得

AI 不是用來取代你的，是用來延伸你的工作時間。

我白天做決策、定方向、處理需要人類判斷的事情。
AI 晚上做執行、研究、維護、那些需要時間但不需要創意的工作。

**一個人 + 一群 AI = 24 小時不停運轉的團隊。**
